v1 — AI launcher (months 1–6)

Android app. Prove the loop. Get 500 real users. Ship nothing you can't rely on.

Weeks 1–2

Contracts + spec

  • Lock 3 workflows only
  • Define capability schema
  • Write approval matrix
  • Pick 5–8 first apps
  • One-page product spec

Weeks 3–4

Shell UI

  • Custom launcher (HOME role)
  • Context surface — no icon grid
  • Universal command bar
  • Task feed skeleton
  • Hardcoded fake flows to test UX

Weeks 5–6

Agent core

  • Intent classifier
  • Planner + task state machine
  • Approval engine (4 levels)
  • Action log — immutable receipts
  • Deterministic first, no LLM yet

Weeks 7–8

Model routing

  • Provider interface (classify, draft, plan)
  • Local model for short private tasks
  • Cloud fallback for hard reasoning
  • No provider names in UI
  • Cache layer for repeat tasks

Weeks 9–10

Capabilities

  • Calendar + reminders
  • Notification triage + drafts
  • Messaging send (with approval)
  • One commerce flow (rides or food)
  • Intents + deep links first, AppFunctions later

Weeks 11–12

Private beta

  • 20–50 real testers
  • Measure: cmds/day, completion rate
  • Measure: fallback rate (target <20%)
  • Fix trust + reliability first
  • Identify top 20 failure patterns

Phase 1 milestones

Launcher lives as default for 10 min without user confusionWeek 4
3 workflows running end-to-end with approvalWeek 10
500 DAU, 60%+ task completion, <20% fallbackMonth 6

v2 — Memory OS (months 7–18)

The OS starts knowing you. This is the moat. No incumbent will build this well due to regulatory exposure.

Months 7–9

Personal knowledge graph

  • On-device graph DB (people, places, projects)
  • STM: last 48h context (hot)
  • MTM: last 30 days (warm, compressed)
  • LTM: preferences, patterns, relationships
  • Full audit UI — user sees everything OS knows

Months 9–12

Context surface v2

  • Home renders from memory + calendar + signals
  • No configuration — inferred from behavior
  • Cold start onboarding (explicit seed questions)
  • Relationship graph surfaces on incoming comms
  • Project awareness across apps

Months 12–15

Proactive layer

  • Interrupt cost model (state-aware)
  • Surfaces only when cost of silence > interrupt cost
  • Morning brief, transition summaries
  • Cross-thread pattern detection
  • User-adjustable sensitivity

Months 15–18

Capability expansion

  • 20+ app integrations
  • Email triage + drafting loop
  • Financial actions (with Level 3 approval)
  • Third-party agent marketplace (early)
  • Enterprise MDM distribution path

Phase 2 milestones

Memory layer live — users can view and edit what OS knowsMonth 9
Home surface requires zero configuration from new usersMonth 12
5,000+ DAU. Retention >60% at 30 days.Month 18

v3 — Native OS (months 19–42)

Move below the launcher. AOSP fork or OEM partnership. The architecture from Blueprint 3 becomes real.

Months 19–24

AOSP fork + agent scheduler

  • Fork AOSP — keep Linux kernel
  • Replace SystemUI with agent-native shell
  • Privileged agent scheduler service
  • Below-the-launcher notification control
  • OEM partnership conversations begin

Months 24–30

Capability-based security

  • Agent contract spec (capability declarations)
  • Fine-grained permission model per agent per action
  • Replace per-app sandbox with agent sandboxing
  • Local-first data — no default telemetry
  • Verifiable audit trail at OS level

Months 30–36

Generative UI runtime

  • Native component renderer from structured intent
  • Context-aware UI density (rush vs. focused vs. idle)
  • Zero permanent apps for top 20 use cases
  • Android compat layer for long-tail apps
  • Developer SDK for skill/agent publishing

Months 36–42

Hardware + silicon path

  • Reference device with OEM partner
  • NPU tuned for on-device model stack
  • Always-on ambient layer (low-power)
  • Cross-device continuity (phone + watch + glasses)
  • Enterprise fleet licensing model

Phase 3 milestones

AOSP fork running on reference hardwareMonth 24
Agent contract spec published — first third-party agentsMonth 30
OEM device ships. 50K+ users on native OS.Month 42

Tech stack — by layer

What you build on, not what you rebuild from scratch.

Surface (v1)Jetpack Compose, Android launcher role (RoleManager), NotificationListenerService
Agent coreKotlin service, task state machine, deterministic planner, action log (append-only SQLite)
CapabilitiesAndroid Intents + deep links first — AppFunctions (Android 16+) when ecosystem matures
Models (v1)Gemini Nano / LiteRT on-device + cloud API fallback (Claude / GPT). No custom training.
Memory (v2)On-device graph DB (e.g. RoomDB + custom schema) + vector index (FAISS or ObjectBox). Never cloud by default.
OS layer (v3)AOSP fork, Linux kernel retained, custom SystemUI replaced, privileged agent scheduler as system service
Security (v3)Capability-based model (Fuchsia-inspired), agent contract spec, per-action permission grants

Risks and how to kill them

Know these before you start, not after they hit.

Launcher switching inertia

Notification triage must be so good that people change launchers just for it. That's the wedge action, not general AI features.

High

AppFunctions ecosystem too thin

Build on intents + deep links first. Treat AppFunctions as a bonus layer, not the foundation. Don't wait for the ecosystem.

High

Trust failure from wrong action

4-level approval model, immutable action log, easy undo. First broken trust event kills the product permanently.

High

Google closes the distribution surface

Enterprise/MDM path as parallel track. A fleet of 10K managed devices beats 100K consumer downloads at this stage.

Medium

Latency makes it feel worse than typing

Local-first for all classification + short drafts. Cloud only for reasoning. Cache aggressively. Target <1.5s for common tasks.

Medium

"Cool demo, no daily habit"

Own the notification + messaging loop. Daily pain, daily use. Everything else is features until this habit is locked.

Medium

Cold start — empty memory is useless

Explicit structured onboarding (10 questions that seed the knowledge graph). Value on day 1 can't require 30 days of observation.
